# frontend-design

Personal skill (plain skill, not a plugin) for distinctive, intentional frontend
visual design, and for stripping the "AI made this" look out of UI.

The judgment-first core (design lead at a studio: read the brief, ground it in
the subject, spend the boldness in one place) merged with a mechanical anti-slop
layer (the specific production tells, the em-dash ban, the consistency locks,
hero discipline, and a pre-flight checklist). Plus when-to-use-a-real-design-system
guidance, a redesign protocol, the intensity dials, a pattern vocabulary, and
canonical motion skeletons.

Started from Anthropic's official `frontend-design` plugin, then merged with the
best of the `tasteskill` anti-slop skill and expanded into a comprehensive
standalone reference.

## Layout

- `SKILL.md` — the judgment, the process, and the reference index.
- `references/anti-slop.md` — reflexes, production tells, the three locks, hero discipline, measurable restraint.
- `references/pre-flight.md` — the mechanical last-pass checklist before shipping.
- `references/design-systems.md` — brief→system map, honesty rules, install commands, sources.
- `references/redesign.md` — detect mode, audit, preserve IA/SEO/voice, modernisation levers.
- `references/design-vocabulary.md` — directions, the intensity dials, the pattern vocabulary.
- `references/typography.md` — scale, glyphs, serif discipline, pairings, italic descender.
- `references/motion.md` — easing, durations, library choice, forbidden React patterns, GSAP skeletons.
- `references/tokens.md` — token system, dark mode, selector-specificity bug.
- `references/responsive-a11y.md` — breakpoints, touch targets, contrast, focus, pre-ship audit.

Install: clone into `~/.claude/skills/frontend-design`.
